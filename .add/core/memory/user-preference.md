# User Coding Preferences
*このファイルはあなたの好みをAIが学習した結果である。*

## 基本スタイル
- **ロジック**: 早期リターン（Guard Clause）を徹底し、ネストを浅く保つ。
- **命名**: 冗長であっても意図が明確な名前を好む。
- **こだわり**: 「書く開発」より「読む開発」を重視。AIによる95点の成果物を人間が5点分磨き上げるスタイル。

## 技術スタック別の好み
- **Laravel**: Controllerは薄く、ビジネスロジックはServiceまたはActionクラスへ。
- **TypeScript**: 型推論に頼りすぎず、関数の境界（引数と戻り値）には明示的に型を付ける。
